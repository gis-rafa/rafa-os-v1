# AI ARCHITECTURE

## User Asks A Question

The user enters a question, request, reflection, problem, or goal into RAFA OS.

The system first classifies the request by intent:

- decision support
- memory lookup
- planning
- project continuation
- learning
- relationship reflection
- health or ADHD support
- general question

The system does not answer immediately. It first determines what context is required to answer safely and usefully.

## What Context Is Loaded

The AI loads context in a strict priority order:

1. `MASTER-BRAIN.md`
2. Active Context
3. Relevant Memory notes
4. Relevant Knowledge notes
5. Relevant Project notes
6. Recent Inbox notes if the request relates to unresolved thoughts
7. Web search only if local context is not enough or freshness matters

The goal is to load the smallest useful context set, not the largest possible context set.

## How MASTER-BRAIN Is Used

`MASTER-BRAIN.md` is the control plane for the AI.

It defines:

- identity
- mission
- decision rules
- active context
- constraints
- personal operating principles
- AI rules

The AI uses `MASTER-BRAIN.md` to understand how to reason, what to prioritize, what tradeoffs matter, and what boundaries must be respected.

It is not treated as ordinary reference material. It has higher authority than project notes, inbox notes, memory notes, or retrieved knowledge.

When there is a conflict, the AI follows this order:

1. explicit user instruction in the current conversation
2. safety and privacy constraints
3. `MASTER-BRAIN.md`
4. active project context
5. memory and knowledge notes
6. general model knowledge

## How Memory Is Searched

Memory contains durable personal context.

Examples:

- preferences
- recurring patterns
- decisions
- relationship context
- ADHD patterns
- health constraints
- long-term goals
- personal rules

Memory search should be used when the question depends on Rafa's history, preferences, commitments, or repeated patterns.

The future system should search memory by semantic relevance, recency, and importance. It should retrieve a small number of high-confidence notes and ignore low-relevance matches.

Memory should not be blindly trusted. If a memory is old, uncertain, emotionally sensitive, or contradicted by newer context, the AI should say so.

## How Knowledge Is Searched

Knowledge contains reusable learning material and domain references.

Examples:

- GIS concepts
- master's notes
- career research
- learning plans
- frameworks
- technical references
- personal branding ideas

Knowledge search should be used when the question depends on facts, frameworks, study material, or reusable domain understanding.

The system should retrieve knowledge separately from memory because knowledge answers "what is true or useful in the domain," while memory answers "what matters about Rafa."

## When Web Search Is Used

Web search is used only when:

- the answer depends on current information
- local knowledge is missing or outdated
- the user asks for latest information
- external facts need verification
- career, market, tool, or technology information may have changed

Web search should not replace RAFA OS memory.

The AI should first check whether the question is personal, local, or already answerable from RAFA OS. If yes, web search is unnecessary.

When web search is used, the AI should cite sources, prefer primary or reliable sources, and clearly separate external facts from personal interpretation.

## Confidence Check

Before generating an answer, the AI must decide:

- Do I have enough information?
- Do I need to ask a follow-up question?
- Do I need memory?
- Do I need knowledge?
- Do I need web search?

The AI should never guess when confidence is low.

If the request is personal, ambiguous, high-impact, or dependent on missing context, the AI should pause and ask a focused follow-up question. If the request depends on Rafa's past, it should use memory. If it depends on reusable domain understanding, it should use knowledge. If it depends on current external facts, it should use web search.

The AI should proceed only when it has enough context to be useful, honest, and safe.

## How The Final Answer Is Generated

The final answer is generated by combining:

- the user's immediate request
- `MASTER-BRAIN.md` rules and constraints
- active context
- relevant memory
- relevant knowledge
- web search results if used

The AI should answer in a way that reduces cognitive load.

The final answer should:

- be direct
- respect Rafa's decision rules
- identify tradeoffs when needed
- give concrete next actions when useful
- avoid unnecessary complexity
- avoid pretending uncertainty is certainty
- preserve Rafa's control over meaningful decisions

For personal decisions, the AI should not simply optimize for speed or productivity. It should consider long-term goals, relationships, health, ADHD, learning, career, and emotional state as one connected system.
